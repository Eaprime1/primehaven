```bash

## Integration with Commit Hook

The `prepare-commit-msg` hook automatically adds prime metadata to commits when:
- Branch name includes `prime-N` pattern
- Commit is not a merge or squash

Metadata added (as comments, hidden in normal git log):
```

# Branch: prime-7/user-auth
# Timestamp: 2026-02-08T23:45:00-05:00

## Philosophy

### Why Prime Progression for PRs?

**Traditional PRs focus on WHAT changed**
- Files modified
- Lines added/removed
- Tests passing

**Prime Progression PRs capture WHY and HOW**
- Consciousness state of the work
- Wisdom gained during development
- Heritage being transmitted
- Growth trajectory

### The 7-Prime Principle (Sacred Completion)

Most PRs should target **7-Prime**:
- Feature is whole (not perfect)
- Ready for integration (not endless refinement)
- Creates completion so next octave can begin
- Sabbath principle: know when to rest

**Avoid:**
- Perfectionism (holding at 6-state forever)
- Premature PR (claiming 7 when at 5)
- Over-extension (7‚Üí19 jump without intermediate primes)

### The 17-Prime Vision (Heritage Crystallization)

Framework PRs should target **17-Prime**:
- Wisdom compressed through experience + reflection + pressure
- Transmittable across contexts and time
- Activates understanding in others
- Working for people not yet born

**This is rare and precious** - most work won't reach 17-Prime, and that's okay. When it does, mark it clearly.

## Best Practices

### 1. Be Honest About Prime State
- Don't claim 7-Prime if work isn't complete
- Don't claim 17-Prime if it's just documentation
- Use lower primes for incremental work

### 2. Capture Sparkles in Real-Time
- Keep `sparkles.txt` during development
- Note "aha!" moments as they happen
- Don't reconstruct later (loses authenticity)

### 3. Use Criteria as Guide, Not Gate
- Completion criteria help assess readiness
- Not a checklist to fake completion
- If unsure, choose lower prime

### 4. Track Your Progression
- View history regularly
- Notice your patterns
- Celebrate 17-Prime moments

### 5. Let Work Find Its Prime
- Don't force work to specific prime
- Let natural state emerge
- Be okay with 3-Prime and 5-Prime work
