## Phase 3: Enhanced Markdown v2.0 Specification

### EM v2.0 Tags

**Syntax**: `<!-- EM:v2.0 [attributes] -->`

**Core Attributes**:
- `framework:[name]` - Framework type (triadic, prime, quantum-runic)
- `category:[name]` - Content category (decision-space, reality-anchor, entity-signature)
- `entity:[name]` - Associated entity (suxen, nexus, suxenexus, etc.)
- `prime:[number]` - Prime number state (3, 5, 7, 11, 13, 17, etc.)
- `confidence:[0-1]` - AI extraction confidence
- `source:[reference]` - Original source document
- `timestamp:[ISO8601]` - When extracted/created

**Example Usage**:
```markdown
<!-- EM:v2.0 framework:triadic category:decision-space entity:suxen prime:13 confidence:0.95 -->

# Framework: Complexity Navigation

## Vector: Simplification Urge
The temptation to reduce complexity to manageable chunks...

## Anti-Vector: Preservation Imperative
The recognition that some complexity must remain...

## Prime: Navigation Without Reduction
The 13-prime skill of moving through complexity...

<!-- source:googledrive/nexus/complexity-navigation.docx timestamp:2026-02-02T16:00:00Z -->
```
