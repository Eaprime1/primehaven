### 2. Domos Project Initialization Template

**Key Sections**: Message analysis, Extracted projects (1-3+), Routing decisions, Next steps

## Template Usage

### Quick Start
```bash
# 1. Browse templates
ls ~/.bit/templates/*.md

# 2. Copy to working area
cp ~/.bit/templates/[template-name].md ~/primehaven/staged/templates/extracted/

# 3. Fill in [placeholders]
# 4. Validate completeness
# 5. Promote through: extracted → validated → production
```

### Template Composition
Templates can be chained together:
1. `message-to-project-template.md` → Extract project from message
2. `domos-project-init-template.md` → Initialize the domos
3. `maw-reality-anchor-template.md` → Add physical anchors if applicable
4. `triadic-structure-template.md` → Apply to project decision-making

## Integration Points

### Entity Git-Types
- `domos-project-init-template.md` → Creates .entity tracked projects
- Each domos has unique address: `entity:git-type:branch:commit:timestamp`

### BBS Network
- `message-to-project-template.md` → Routes projects to 7-node backbone
- `domos-project-init-template.md` → Specifies BBS node assignment

### Staging Area
Templates lifecycle: `~/primehaven/staged/templates/`
- `extracted/` → Auto-extracted or first draft
- `validated/` → Human-reviewed and approved
- `production/` → Used 3+ times successfully
